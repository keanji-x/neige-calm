## What changed

Commit 1 (`e88f4312`) adds sibling `codex-worker` and `terminal-worker` provider adapters and routes dispatcher worker request arms through `OperationRuntime::start`. The adapters preserve worker card role/deletability, payload bookkeeping, prompt rendering, cwd fallback, post-renderer `CardAdded` emission, and reap-before-delete compensation.

Commit 2 (`f95f9c76`) deletes the dispatcher-local worker spawn/idempotency/rollback stack: `spawn_codex_worker`, `spawn_terminal_worker`, `rollback_orphan_worker`, `RollbackOutcome`, `find_card_by_idempotency_key_tx`, `recently_seen`, and `RecentlySeenGuard`. Dispatcher docs/tests now describe operations-table idempotency and adapter compensation.

## Why this scope

Option A from exploration is implemented: `codex-worker` and `terminal-worker` are sibling operation kinds, not a mode discriminator on the existing create adapters. This keeps the worker-specific prompt/bookkeeping/compensation behavior isolated while preserving the existing public `codex-create` / `terminal-create` routes.

Legacy dispatcher-only unit tests for `RecentlySeenGuard` and card-payload idempotency collision were deleted because the invariant is now owned by `OperationRuntime` / `SqlxOperationRepo`; framework coverage remains in `operation::tests::same_idempotency_key_different_hash_conflicts`. Dispatcher integration tests were rewritten where the old same-key retry contract conflicted with operation-table idempotency.

## Externally-visible semantic change

- Duplicate-failed retry now sticks at `failed`: operations-table idempotency reuses the same operation row instead of allowing a later retry with the same key to succeed. The rewritten `tests/dispatcher.rs` duplicate-failed retry coverage pins this behavior.
- Duplicate `*.Requested` for an already-failed key may emit a second `TaskFailed`: dispatcher maps `OperationOutcome::Failed` through the retry loop into another `Event::TaskFailed`. This is accepted for this PR; a follow-up can decide whether suppressing duplicate failure broadcasts is worth the extra state.
